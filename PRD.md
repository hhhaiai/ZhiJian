## 一、产品概述

### 1.1 产品定义

**需求质检台** 是一款帮助产品经理提升需求文档质量的辅助工具。

### 1.2 核心价值

> **让产品经理在提交需求之前，先经过 AI 的"辅助检查"，把能发现的问题先标出来，减少开发过程中的反复确认。**

### 1.3 产品口号

```
写需求 → AI 辅助检查 → 补充完善 → 高质量交付
```

## 二、用户角色与权限

### 2.1 角色定义

| 角色 | 权限 | 说明 |
|------|------|------|
| 产品经理 | 全部 | 录入需求、AI 检查、编辑、删除、状态变更 |
| UI设计师 | 查看 | 查看需求、查看设计图链接 |
| 开发 | 查看 | 查看需求 |
| 测试 | 查看 | 查看需求 |
| 客服 | 录入 | 提交反馈（可选） |

### 2.2 MVP 阶段说明

- MVP 阶段不实现登录权限控制
- 所有访问者默认具有产品经理权限
- 未来可通过配置实现权限区分

### 2.3 扩展：用户系统（V2.0）

| 功能 | 说明 |
|------|------|
| 用户注册 | 支持用户名+密码注册 |
| 用户登录 | 登录后可查看自己的创建记录 |
| 权限控制 | 根据角色控制操作权限 |
| 个人设置 | 修改昵称、头像、密码 |

**入口**：应用首页

**功能**：
- 展示所有需求卡片

- 搜索：按标题关键词搜索
- 新建：点击跳转新建页面
- 刷新：重新获取最新数据

```
┌────────────────────────────────────────────────────────┐
│  状态: [全部 ▼]  优先级: [全部 ▼]     [+ 新建需求]      │
```

| 筛选字段 | 选项 |

## 五、交互流程

### 5.1 新建需求流程

```
1. 用户点击 [+ 新建需求]
2. 跳转至需求录入页
3. 用户填写：
   - 需求标题（必填）
   - 需求描述（必填，使用 Markdown）
   - 设计图链接（可选）
   - 优先级（必选）
   - 需求类型（必选）
4. 自动保存触发（每 30 秒）
5. 用户选择：
   a. 点击 [保存草稿]
      - 保存为"草稿"状态
      - 跳转回列表页
      - 显示 Toast: "已保存为草稿"
   b. 点击 [提交检查]
      - 校验必填项
      - 保存为"待检查"状态
      - 触发 AI 检查
      - 跳转到详情页，显示 AI 检查结果
```

**自动保存逻辑**：
- 定时保存：每 30 秒
- 页面离开保存：unload 事件
- 保存到本地存储（LocalStorage）
- 保存成功后显示"已自动保存"

### 5.2 AI 检查流程

```
1. 用户在详情页点击 [提交检查] 或 [重新检查]
2. 显示 loading 状态："AI 正在检查..."
3. 后端调用 AI 模型 API
4. AI 返回检查结果
5. 后端保存检查结果到需求记录
6. 前端展示 AI 检查结果面板

异常情况处理：
- AI 服务不可用
  → 显示错误提示："AI 服务暂时不可用，请稍后重试"
  → 提供 [重新检查] 按钮

- AI 返回格式错误
  → 显示错误提示："检查结果解析失败"
  → 提供 [重新检查] 按钮

- 请求超时（30秒）
  → 显示错误提示："检查超时，请重试"
  → 提供 [重新检查] 按钮

- 未配置 AI
  → 跳转到 AI 配置页面
  → 提示："请先配置 AI"
```

### 5.3 确认完善流程

```
1. 用户查看 AI 建议
2. 根据建议修改需求描述（可选）
3. 点击 [确认完善]
4. 需求状态变更为"确认完善"
5. 显示 Toast: "已确认完善"
```
